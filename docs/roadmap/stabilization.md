## Модуль 0.1: Критические баги (P0)

### 0.1.1: Удаление технологий не обновляет UI
- **Файл:** `frontend/src/pages/TechnologiesPage.tsx`
- **Проблема:** `deleteTech()` / `deleteArtifact()` вызываются напрямую без `useMutation` и без `invalidateQueries`. Удалённый элемент остаётся на экране до перезагрузки.
- **Решение:** Заменить на `useMutation` с инвалидацией `['techs', universeId]` и `['artifacts', universeId]`.

### 0.1.2: Graph Page не обновляет view «Space»
- **Файл:** `frontend/src/pages/GraphPage.tsx`
- **Проблема:** При удалении/добавлении связей инвалидируется `['graph', universeId]`, но не `['graph-space', universeId]`.
- **Решение:** Добавить `invalidateQueries({ queryKey: ['graph-space', universeId] })`.

### 0.1.3: Race condition в BookEditorPage при смене act
- **Файл:** `frontend/src/pages/BookEditorPage.tsx`, ~строка 1004
- **Проблема:** `deleteOutlineMutation.mutate()` / `moveOutlineMutation.mutate()` вызываются без `await`. Быстрые клики создают параллельные мутации.
- **Решение:** Использовать `.mutateAsync()` с `await` + disable селекта пока мутация не завершена.

### 0.1.4: Позиции графа сохраняются с тихим фейлом
- **Файл:** `frontend/src/components/KnowledgeGraph.tsx`, строка 119
- **Проблема:** `.catch(() => {})` — ошибки проглатываются, пользователь не знает, что позиция не сохранена.
- **Решение:** Показать `toast.error()` при ошибке сохранения.

## Модуль 0.2: Завершение незавершённых функций (P0)

### 0.2.1: Пустые состояния для всех страниц
- **Затронутые страницы:** TechnologiesPage, FactionsPage, ConceptArtPage, DraftsPage
- **Проблема:** Нет компонента `EmptyState` — пользователь видит пустую таблицу без подсказки что делать.
- **Решение:** Добавить `EmptyState` с CTA-кнопкой («Создать первую технологию», «Добавить первую фракцию» и т.д.).

### 0.2.2: Валидация форм
- **Затронутые страницы:** TechnologiesPage, FactionsPage, ArtifactsPage
- **Проблема:** Нет клиентской валидации кроме HTML `required`. `influence_level` не проверяет bounds.
- **Решение:** Добавить `validateTechnology()`, `validateArtifact()`, `validateFaction()` в `utils/validation.ts`.

### 0.2.3: Типизация `any` в ключевых местах
- **Затронутые файлы:**
  - `SpacePage.tsx` — 17+ `any`
  - `DevelopUniversePage.tsx` — 12 `any`
  - `LocationsPage.tsx` — 12 `any`
  - `CharactersPage.tsx` — 10 `any`
  - `BookEditorPage.tsx` — 5 `any`
- **Проблема:** Нет type safety, тихие runtime-ошибки.
- **Решение:** Определить интерфейсы и заменить `any` на типизированные состояния.

### 0.2.4: FactionsPage — `editingFaction` и `generatedFactions` как `any`
- **Файл:** `frontend/src/pages/FactionsPage.tsx`
- **Проблема:** Форма работает с `any`, может отправить malformed данные.
- **Решение:** Определить `FactionFormData` интерфейс.

## Модуль 0.3: Исправление UX-проблем (P1)

### 0.3.1: Заменить `alert()` на `toast.error()`
- **Файл:** `frontend/src/components/chat/TimePickerModal.tsx`, строки 83, 87
- **Проблема:** Нативный `alert()` блокирует UI, не соответствует теме.
- **Решение:** Заменить на `toast.error()`.

### 0.3.2: Заменить `confirm()` на кастомный модал
- **Количество:** ~25 вызовов `confirm()` по 17 файлам
- **Проблема:** Нативный диалог блокирует main thread, не стилизуется, выглядит чужеродно в dark mode.
- **Решение:** Создать `ConfirmModal` компонент, заменить все вызовы.

### 0.3.3: API-клиент логирует ошибки в production
- **Файл:** `frontend/src/api/client.ts`, строки 35, 39
- **Проблема:** `console.error()` для каждого 4xx/5xx, засоряет консоль.
- **Решение:** Обернуть в `if (import.meta.env.DEV)` или логировать только 5xx.

### 0.3.4: Optimistic update edge case в DraftsPage/NotesPage
- **Файл:** `frontend/src/pages/DraftsPage.tsx`, строки 57-78
- **Проблема:** При ошибке сервера пользователь видит запутанное сообщение «сервер не сохранил, изменения видны локально».
- **Решение:** Автоматический rollback при ошибке или автоматический retry.

## Модуль 0.4: Обработка ошибок бэкенда (P1)

### 0.4.1: Централизация обработки ошибок API
- **Проблема:** Разные эндпоинты по-разному обрабатывают ошибки — где-то `HTTPException`, где-то голый `raise`.
- **Решение:** Создать единый exception handler + middleware для консистентного формата ошибок.

### 0.4.2: RAG — обработка недоступности ChromaDB
- **Проблема:** Если ChromaDB упал, запрос к чату падает.
- **Решение:** Обернуть вызовы RAG в `try/except`, при ошибке — пустой RAG-блок + warning log.

### 0.4.3: Поиск — таймаут на больших вселенных
- **Проблема:** Поиск по всем категориям может зависнуть на больших данных.
- **Решение:** `asyncio.wait_for()` с таймаутом, возврат частичных результатов.

### 0.4.4: Кэш контекста — сброс при смене настроек
- **Проблема:** При отключении RAG старый контекст может отдаваться из кэша.
- **Решение:** Добавить `enable_rag` в ключ кэша.
