# Рекомендации по улучшению ARCHON

Краткий список того, что можно доработать для контекста, поиска, RAG и UX.

---

## Сделано в этой сессии

- **Переключатель «Все черновики в контекст»** добавлен на страницу настроек (Расширенные → Параметры RAG и контекста). Раньше опция была только в БД.

---

## 1. Контекст и RAG

| Улучшение | Описание | Сложность |
|-----------|----------|-----------|
| **Вынести лимиты в настройки** | `BUDGET_BASE`, `BUDGET_CREATOR_CHARS`, `BUDGET_MENTIONS`, бюджет технологий/артефактов (4000) сейчас зашиты в коде. Имеет смысл часть из них (например лимит списка технологий) вынести в `system_settings` или в `prompt_settings`, чтобы настраивать без правки кода. | Низкая |
| **RAG: эмбеддинги по умолчанию** | Сейчас RAG без `RAG_EMBEDDING_MODEL` использует встроенные эмбеддинги ChromaDB (англоязычные). Для русского лучше задать модель в `.env` (например `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`) и описать это в README / .env.example. | Низкая |
| **Дедупликация контекста** | При сборе контекста из поиска и RAG один и тот же фрагмент может попасть и из поиска по заметкам, и из RAG. Можно дедуплицировать по `entity_type`+`entity_id` или по хэшу текста и не дублировать в промпте. | Средняя |
| **Переиндексация заметок** | Для уже существующих заметок чанковая индексация в RAG обновляется только при сохранении. Добавить в API или админку действие «Переиндексировать все заметки/черновики вселенной» (и при желании технологии/артефакты). | Средняя |

## 2. Поиск

| Улучшение | Описание | Сложность |
|-----------|----------|-----------|
| **Поиск по фракциям** | Сейчас в `SearchService` нет раздела «фракции». Добавить `_search_factions` и включить `factions` в результаты и в `section_labels` в промпте — тогда запросы про фракции будут подтягивать релевантные карточки. | Низкая |
| **Ранжирование по релевантности** | Сейчас по категориям возвращается просто список. Можно ввести простой счёт (сколько слов запроса совпало, в скольких полях) и сортировать по убыванию, чтобы самые релевантные шли первыми и чаще попадали в лимит контекста. | Средняя |
| **Подсветка совпадений в сниппетах** | `_highlight_match` даёт один фрагмент. Для нескольких вхождений можно возвращать несколько сниппетов или помечать совпадения тегом/маркером в тексте для UI. | Низкая |

## 3. Фронтенд и UX

| Улучшение | Описание | Сложность |
|-----------|----------|-----------|
| **Предпросмотр контекста перед отправкой** | Перед отправкой сообщения в чат вызывать поиск по текущему тексту (`/api/universes/{id}/search?q=...`) и показывать блок вида: «По запросу будут использованы: заметка X, технология Y, глава Z». Так пользователь видит, что попадёт в контекст. | Средняя |
| **Подсказка по бюджету** | В настройках «Бюджет черновиков» и «Размер окна контекста» добавить краткую подсказку: что именно ограничивается (символы черновиков / токены окна) и как это связано с стоимостью/лимитами модели. | Низкая |
| **Чат: индикатор «контекст загружен»** | После формирования промпта можно (опционально) показывать короткую сводку: «В контексте: N персонажей, поиск: X записей, RAG: Y фрагментов» — чтобы было понятно, что контекст собран. | Средняя |

## 4. Надёжность и производительность

| Улучшение | Описание | Сложность |
|-----------|----------|-----------|
| **Обработка ошибок RAG** | При недоступности ChromaDB или падении эмбеддингов запрос к чату не должен падать. В местах вызова RAG обернуть в try/except и при ошибке подставлять пустой RAG-блок и логировать предупреждение. | Низкая |
| **Таймаут поиска** | Поиск по всем категориям (включая технологии, артефакты, главы) может быть тяжёлым на больших вселенных. Ограничить время выполнения (например asyncio.wait_for) и при таймауте возвращать частичные результаты или пустой блок поиска. | Низкая |
| **Кэш контекста: сброс при изменении настроек** | Сейчас ключ кэша учитывает `draft_budget` и `include_all_drafts`. При смене других настроек (например отключение RAG в системе) старый контекст может ещё отдаваться из кэша. Добавить в ключ кэша, например, `enable_rag` и при необходимости хэш настроек промпта. | Низкая |

## 5. Структурирование данных

| Улучшение | Описание | Сложность |
|-----------|----------|-----------|
| **Чанкование по заголовкам (Markdown)** | Для заметок/черновиков в Markdown можно резать по `##` / `###` и класть в RAG отдельные чанки с метаданными «заголовок». Тогда в контекст попадёт только нужный раздел, а не весь документ. | Средняя |
| **Краткое описание заметки** | Добавить поле «тезисы» / «краткое содержание» у заметки. В контекст по умолчанию подставлять его, а полный текст — только при попадании в поиск/RAG. | Средняя |
