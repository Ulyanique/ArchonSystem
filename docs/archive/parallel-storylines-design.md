# Параллельные сюжетные линии: проект и интеграция

## Текущее состояние

- **Книга** пишется **линейно**: главы принадлежат вселенной, порядок задаётся полем `chapter_number`.
- Модели: `Chapter` (universe_id, chapter_number, title, content, summary, …), `OutlineItem` (привязка к главе), `TimelineEvent` (опционально chapter_id), `SceneBeat` (внутри главы).
- Экспорт, чат, RAG, поиск используют один и тот же порядок: по `chapter_number`.

## Цель

Поддержать **несколько сюжетных линий** (например, линия Алисы, линия Боба, общая линия), чтобы:
- главы можно было группировать по линиям и нумеровать внутри линии;
- при экспорте и отображении можно было выбирать порядок: по линиям подряд (все главы линии 1, потом линии 2) или **чередование** (глава 1 линии A, глава 1 линии B, глава 2 линии A, …);
- сюжетная линия могла быть привязана к персонажу (опционально), для удобства и фильтрации.

## Модель данных

### 1. Новая сущность: Storyline (сюжетная линия)

| Поле | Тип | Описание |
|------|-----|----------|
| id | int | PK |
| universe_id | int | Вселенная |
| title | string | Название (например «Линия Алисы», «Арка злодея») |
| description | text | Краткое описание линии |
| sort_order | int | Порядок линии в списке (для UI и порядка «сначала все главы линии 1») |
| main_character_id | int, nullable | FK → characters.id, опционально «главный» персонаж линии |

Хранится в **Universe DB** (как Chapter), не в Master.

### 2. Изменения в Chapter

| Поле | Тип | Описание |
|------|-----|----------|
| storyline_id | int, nullable | FK → storylines.id. **null** = глава «общей»/основной линии (текущее поведение) |
| storyline_order | int, default 0 | Порядок главы **внутри** своей сюжетной линии |
| reading_order | int, nullable | Глобальный порядок чтения при экспорте. Если задан — сортировка по нему; иначе по (storyline.sort_order, storyline_order) |

**Обратная совместимость:**
- Все существующие главы остаются с `storyline_id = null`; порядок по-прежнему задаётся `chapter_number`.
- Можно трактовать «главы без линии» как одну линейную книгу и не заводить запись Storyline, пока пользователь не создаёт линии.

**Нумерация:**
- Для глав с `storyline_id = null`: как сейчас — отображаем и сортируем по `chapter_number`.
- Для глав с `storyline_id != null`: внутри линии показываем «Глава N» по `storyline_order` (или по `chapter_number`, если решим дублировать). В списке/экспорте порядок: либо по `reading_order`, либо по (storyline.sort_order, storyline_order).

### 3. Связи

- **Storyline** → Character (main_character_id, опционально).
- **Chapter** → Storyline (storyline_id, опционально).
- OutlineItem, TimelineEvent, SceneBeat продолжают ссылаться на Chapter; дополнительных полей не требуется.

## Интеграция в систему

### Репозитории и API

- **Storyline:** CRUD репозиторий и API: `GET/POST /api/universes/{id}/storylines`, `GET/PUT/DELETE .../storylines/{sid}`.
- **Chapters:** при создании/редактировании главы передавать `storyline_id`, `storyline_order`, `reading_order`. Список глав: поддержка фильтра `?storyline_id=` и сортировки по выбранному режиму (по линии, по reading_order).

### Порядок глав

- Единая функция получения «упорядоченного списка глав» для экспорта и отображения:
  - если у вселенной не используется линии (все storyline_id = null) — сортировка по `chapter_number` как сейчас;
  - иначе: если задан `reading_order` — по нему; иначе сортировка по (storyline.sort_order, chapter.storyline_order), при этом главы без линии (storyline_id = null) можно считать одной виртуальной линией с минимальным sort_order.
- В **ChaptersPage** (фронт): переключатель/табы «Все главы» / по линиям; возможность задать `reading_order` (например, drag-and-drop общий список для чередования).

### Контекст чата и RAG

- Логика контекста и RAG не обязана знать про линии: в контекст попадают выбранные главы/заметки и релевантные по поиску/RAG. При необходимости можно добавить фильтр «только главы линии X» в настройках чата или в запросе — по желанию.
- **Включённые в контекст главы** (уже реализовано): пользователь по-прежнему выбирает главы по списку; список может быть сгруппирован по сюжетным линиям.

### Поиск и умный контекст

- Поиск по главам (SearchService) возвращает главы; при отображении можно показывать название сюжетной линии. Группировка по линиям в UI — без изменения контракта поиска.
- Контекст «Помощника Создателя» (context_manager): можно при желании добавлять в блок контекста краткий список сюжетных линий и привязку глав к линиям, чтобы ИИ знал структуру.

### Экспорт (Markdown / DOCX)

- Использовать единый упорядоченный список глав (по reading_order или по линии).
- В заголовках глав в экспорте можно добавлять префикс линии, например: `### Глава 2 (Линия Алисы): Название` или оставить просто «Глава N: Название» с учётом общей нумерации.

### Ай-критик и генерация

- Ай-критик и генерация текста по главам работают с главами как раньше; при передаче списка глав в промпт можно добавлять метку линии для ясности.

### Таймлайн и план (Outline)

- Таймлайн и план привязаны к главам (chapter_id). Дополнительных полей не нужно; при отображении события/пункта плана можно показывать название линии главы.

## UI (кратко)

- **Раздел «Сюжетные линии»** в меню вселенной: список линий, создание/редактирование, привязка к персонажу, sort_order.
- **Страница глав:** фильтр/табы по сюжетной линии; при создании/редактировании главы — выбор линии и номер внутри линии; опционально — общий список с drag-and-drop для `reading_order` (чередование).
- **Экспорт:** опция «Порядок: по линиям» / «Чередование (по reading_order)» если задан.

## Этапы внедрения

1. **Модели и миграции:** таблица `storylines`; в `chapters` добавить `storyline_id`, `storyline_order`, `reading_order`.
2. **Репозиторий и API сюжетных линий** + обновление API глав (схемы, валидация).
3. **Логика порядка глав:** одна функция в репозитории/сервисе для «глав в порядке чтения»; подключить в экспорт и в API списка глав.
4. **Фронт:** страница/секция сюжетных линий; на странице глав — выбор линии при создании/редактировании, фильтр по линии, при необходимости — редактор reading_order.
5. **Дорастки:** отображение линии в контексте чата/RAG, префикс линии в экспорте, подсказки в Ай-критике.

После этого книга сможет вестись и линейно (как сейчас), и в виде параллельных сюжетных линий с гибким порядком чтения.
