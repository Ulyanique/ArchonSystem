# Сервисы

Сервисы расположены в `backend/app/services/`. Они содержат всю бизнес-логику, независимую от HTTP-слоя.

## LLMService

**Файл:** `llm.py`

Единый интерфейс к 4 LLM-провайдерам с автоматическим fallback.

### Провайдеры

| Провайдер | Класс | Тип | URL |
|-----------|-------|-----|-----|
| Ollama | `OllamaProvider` | Локальный | `http://localhost:11434` |
| DeepSeek | `DeepSeekProvider` | Облачный | API DeepSeek |
| OpenRouter | `OpenRouterProvider` | Агрегатор | API OpenRouter |
| RouterAI | `RouterAIProvider` | Альтернативный | API RouterAI |

### Ключевые методы

```python
async def chat(messages: list, system: str, stream: bool, model: str, ...)
    # Основной метод генерации текста
    # Поддерживает streaming (SSE) и non-streaming режимы
    # Автоматический retry при ошибках

async def generate_image(prompt: str, width: int, height: int, ...)
    # Генерация изображений через 4 провайдера:
    # OpenRouter (Gemini 2.5 Flash Image), Cloudflare Workers AI,
    # Pixazo (Flux 1 Schnell / Flux 2 Pro), Whisk (Google Labs)
```

### retry-логика

- Настраиваемое количество попыток
- Экспоненциальная задержка между попытками
- Автоматический fallback на следующий провайдер

## ContextManager

**Файл:** `context_manager.py`

Управление контекстом чата с персонажем.

### Функции

- **Кэширование с TTL**: кэш контекста для уменьшения повторных запросов
- **Бюджетирование символов**: настраиваемые лимиты для каждого блока контекста
- **Обнаружение упоминаний**: парсин текста на `[[type:id]]` маркеры
- **Фильтрация по знаниям**:过滤 событий по уровню доступа персонажа
- **Prefix Caching**: стабильный порядок системного промпта для Ollama

### Структура системного промпта

```
[Статичный мир]        → мир, настройки, стиль
[Персонаж]            → имя, характер, речь, знания
[Динамические данные]  → RAG-найденные фрагменты, упоминания
```

## CharacterContext

**Файл:** `character_context.py`

Фильтрация информации по уровням доступа персонажа.

### Уровни знаний

| Уровень | Описание |
|---------|----------|
| `none` | Персонаж ничего не знает |
| `rumors` | Слышал слухи |
| `superficial` | Поверхностное знание |
| `good` | Хорошее знание |
| `complete` | Полное знание |

### Источники знаний

| Источник | Описание |
|----------|----------|
| `participated` | Персонаж участвовал |
| `witnessed` | Персонаж был свидетелем |
| `heard` | Слышал от других |
| `read` | Прочитал в документах |
| `learned_from` | Изучил из другого источника |

### Temporal Filter

Персонаж не знает событий, которые произошли «после» момента диалога.

## AIWriter

**Файл:** `ai_writer.py`

Написание текста глав с поддержкой streaming.

### Режимы

| Режим | Описание |
|-------|----------|
| `from_summary` | Написание из синопсиса главы |
| `continue` | Продолжение существующего текста |
| `by_instruction` | Написание по инструкции автора |

### Формат ответа

Streaming через SSE: каждый токен отправляется как отдельное событие.

## AICritic

**Файл:** `ai_critic.py`

AI-анализ согласованности контента.

### Типы анализа

| Тип | Что анализирует |
|-----|-----------------|
| Character | Согласованность характера, мотивации, речи |
| Location | Детальность описания, атмосфера |
| Chapter | Структура, pacing, согласованность с сюжетом |
| Act | Согласованность акта с outline |
| SceneBeat | Связность сцены с главой |
| Note | Полезность и релевантность заметки |
| Consistency | Общая согласованность книги |

## Search

**Файл:** `search.py`

Полнотекстовый поиск по всем элементам вселенной.

### Источники поиска

- Персонажи (имя, описание, характер)
- Локации (название, описание, атмосфера)
- Главы (название, синопсис, текст)
- Заметки (заголовок, содержимое)
- Цитаты (текст, контекст)
- Фракции (название, описание)
- Таймлайн (название, описание)
- Wiki-статьи (заголовок, содержимое)
