# Архитектура системы

## Обзор

ARCHON — это fullstack-приложение с чистой многослойной архитектурой:

```
┌─────────────────────────────────────────────────────┐
│                  Фронтенд (React)                    │
│  Страницы → Компоненты → API-клиент → Zustand       │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP / SSE (streaming)
┌──────────────────────┴──────────────────────────────┐
│                Бэкенд (FastAPI)                      │
│  API Routes → Services → Repositories → Models       │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────┴──────────────────────────────┐
│           Базы данных (SQLite + ChromaDB)            │
│   Master DB ─── Universe DB (per universe)          │
│   ChromaDB ─── Векторный поиск (RAG)                │
└─────────────────────────────────────────────────────┘
```

---

## Архитектурные слои

### 1. Presentation Layer (Фронтенд)

- **React 18 + TypeScript** — типизированные компоненты
- **Vite 5** — быстрая сборка и HMR
- **React Router v6** — вложенные маршруты (layout pattern)
- **TanStack Query** — кэширование, инвалидация, оптимистичные обновления
- **Zustand** — лёгкое глобальное состояние (выбранная вселенная, сайдбар)
- **Tailwind CSS** — утилитарные стили с кастомными анимациями

**Ключевой принцип:** Страницы оркестрируют, компоненты переиспользуются, API-клиент типизирован.

### 2. API Layer (Бэкенд — `app/api/`)

30 эндпоинт-файлов, каждый отвечает за свою предметную область:
- RESTful маршруты с префиксами `/api/universes/{id}/...`
- Автоматическая регистрация через `routes.py` (импорт из `app/api/`)
- Streaming через Server-Sent Events (SSE) для AI-генерации и чата
- Файловые операции (загрузка обложек, портретов, концепт-артов)

### 3. Service Layer (Бэкенд — `app/services/`)

Бизнес-логика, независимая от HTTP:

| Сервис | Ответственность |
|--------|----------------|
| `LLMService` | Единый интерфейс к 4 LLM-провайдерам + 4 провайдерам изображений |
| `RAGService` | Векторный поиск через ChromaDB, эмбеддинги, чанкинг |
| `ContextManager` | Управление контекстом чата, кэширование, бюджетирование |
| `AIWriter` | Написание текста глав (streaming, 3 режима) |
| `AICritic` | Анализ согласованности (персонажи, локации, главы) |
| `AIGenerator` | Генерация сущностей (персонажи, локации, имена, plot twist) |
| `PromptBuilder` | Сборка системного промпта с шаблонами |
| `TimeService` | Расчёт внутреннего времени (кастомный календарь) |
| `Search` | Полнотекстовый поиск по всем элементам |
| `Export` | Экспорт в Markdown и DOCX |
| `Links` | Граф знаний, анализ связанности |

### 4. Repository Layer (Бэкенд — `app/repositories/`)

Data access layer — CRUD-операции над SQLAlchemy моделями:
- 17 репозиториев, по одному на доменную сущность
- Async-сессии SQLAlchemy
- Транзакционность и обработка ошибок

### 5. Domain Layer (Бэкенд — `app/models/`)

19 SQLAlchemy-моделей, представляющих доменные сущности:
- Universe, Book, Character, Location, Chapter, SceneBeat
- Storyline, TimelineEvent, OutlineItem, Note, Quote, Link
- Faction, ConceptArt, Technology, Artifact
- Galaxy, StarSystem, CelestialBody, ChapterMention
- GraphLayout, SystemSettings, CharacterKnowledge

### 6. Data Layer

**Master DB** (`archon_master.db`):
- `universes` — метаданные вселенных
- `books` — книги/продукты
- `graph_layout` — позиции узлов графа
- `system_settings` — глобальные настройки (LLM, тема, промпты)

**Universe DB** (`data/universes/{id}/universe.db`):
- Каждая вселенная — отдельная база данных
- Полная изоляция данных между вселенными
- Возможность бэкапа/восстановления по отдельности

**ChromaDB** (векторная БД):
- RAG-поиск по главам и заметкам
- Мультиязычные эмбеддинги (sentence-transformers)
- Изоляция по `universe_id`

## Ключевые архитектурные паттерны

### Universe-per-Database

Каждая вселенная имеет собственную SQLite-базу данных. Это обеспечивает:
- **Изоляцию**: данные вселенных не пересекаются
- **Портативность**: вселенную можно скопировать, перенести, заархивировать
- **Безопасность**: удаление вселенной = удаление файла БД
- **Масштабируемость**: можно добавить поддержку PostgreSQL/MySQL в будущем

### LLM Abstraction

`LLMService` абстрагирует 4 провайдера за единым интерфейсом:
- Автоматический fallback при недоступности провайдера
- Единый формат запросов и ответов
- Поддержка streaming (SSE)
- Retry-логика с настраиваемыми попытками

### RAG + Knowledge Filtering

При чате с персонажем контекст формируется в несколько этапов:
1. **RAG-поиск**: извлекаются релевантные фрагменты из ChromaDB
2. **Knowledge Filter**: события фильтруются по уровню доступа персонажа
   - `none` → `rumors` → `superficial` → `good` → `complete`
3. **Temporal Filter**: персонаж не знает событий «из будущего» относительно момента диалога
4. **Budget Capping**: каждый блок контекста имеет лимит символов

### Prefix Caching для Ollama

Системный промпт формируется в фиксированном порядке:
```
[Статичный мир] → [Персонаж] → [Динамические упоминания]
```
Это позволяет Ollama кэшировать неизменяемую часть промпта между запросами.

### Streaming (SSE)

AI-генерация и чат работают через Server-Sent Events:
- Клиент получает токены по мере генерации
- UI обновляется в реальном времени
- Нет блокировки интерфейса

## Модульность

Проект следует принципу **разделения ответственности**:

```
backend/app/
├── api/          → HTTP-маршруты (валидация входных данных, вызов сервисов)
├── services/     → Бизнес-логика (алгоритмы, интеграции с LLM)
├── repositories/ → Доступ к данным (CRUD, запросы)
├── models/       → Доменные сущности (SQLAlchemy)
├── schemas/      → Схемы данных (Pydantic)
└── utils/        → Общие утилиты
```

```
frontend/src/
├── pages/        → Страницы-маршруты (оркестрация)
├── components/   → Переиспользуемые UI-компоненты
├── api/          → API-клиенты (typed Axios)
├── store/        → Zustand stores
├── utils/        → Утилиты
└── types.ts      → TypeScript-типы
```

```
         │
         ▼
┌─────────────────┐     ┌──────────────┐

└────────┬────────┘     └──────────────┘
         │
         ▼
┌─────────────────┐     ┌──────────────┐
│  Repositories   │────▶│  SQLite DB   │  Master + Universe
└────────┬────────┘     └──────────────┘
         │
         ▼
┌─────────────────┐
│   ChromaDB      │  RAG-эмбеддинги
└─────────────────┘
```
