# ARCHON
## Система управления вселенными

ARCHON — это комплексная система для управления и моделирования вселенных, сочетающая мощный бэкенд на Python с современным фронтендом на React/Vite.

### 📁 Структура проекта

```
ARCHON/
├── backend/          # Бэкенд на FastAPI
│   ├── app/          # Основные модули приложения
│   ├── data/         # Работа с данными и базами
│   ├── tests/        # Тесты бэкенда
│   ├── venv/         # Виртуальное окружение (игнорируется)
│   ├── requirements.txt
│   └── .env.example  # Пример конфигурации
├── frontend/         # Фронтенд на React + Vite
│   ├── src/          # Исходный код
│   ├── node_modules/ # Зависимости (игнорируются)
│   ├── package.json
│   └── vite.config.ts
├── backups/          # Резервные копии (игнорируются)
├── design/           # Дизайн-документы (игнорируются)
├── docs/             # Документация (игнорируются)
├── .gitignore        # Игнорируемые файлы
└── README.md         # Этот файл
```

### ⚙️ Технологический стек

**Бэкенд:**
- Python 3.10+ (рекомендуется 3.10 или 3.11)
- FastAPI для REST API
- SQLAlchemy для работы с базами данных
- Pydantic для валидации данных
- ChromaDB для векторных баз данных
- Sentence Transformers для NLP

**Фронтенд:**
- React 18 + TypeScript
- Vite для сборки
- Tailwind CSS для стилизации
- React Flow для визуализации графов
- Zustand для управления состоянием
- TanStack Query для работы с данными

### 🚀 Как запустить

#### Бэкенд
```bash
cd backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
uvicorn app.main:app --reload
```
Для воспроизводимой сборки используйте зафиксированные версии: `pip install -r requirements-pinned.txt`

#### Фронтенд
```bash
cd frontend
npm install
npm run dev
```
Для установки строго по lockfile: `npm ci`

После запуска бэкенда документация API доступна по адресам:
- **Swagger UI:** http://localhost:8000/docs  
- **ReDoc:** http://localhost:8000/redoc

### Переменные окружения (backend)

Скопируйте `backend/.env.example` в `backend/.env` и заполните значения. **Файл `.env` не коммитьте в репозиторий** — в нём хранятся секреты (ключи API).

Основные переменные: LLM-провайдеры (Ollama, DeepSeek, OpenRouter, RouterAI), ключи API, порт. Дополнительно:
- `DEBUG` — при `true` в ответах при ошибках возвращается текст исключения; в production оставьте `false`.
- `CORS_ORIGINS` — список разрешённых источников для CORS через запятую (по умолчанию localhost:5173 и 3000).
- `RAG_EMBEDDING_MODEL` — модель sentence-transformers для эмбеддингов в RAG (векторный поиск по главам и заметкам). По умолчанию используется мультиязычная модель, хорошо подходящая для русского. Можно задать другую (например `intfloat/multilingual-e5-small`) или оставить пустой строкой, чтобы использовать встроенные эмбеддинги ChromaDB (хуже для русского текста).

### RAG и контекст чата

В чате с ИИ используется **RAG** (Retrieval Augmented Generation): по запросу пользователя из векторной базы (ChromaDB) подбираются релевантные фрагменты глав и заметок и подставляются в контекст. Для качественного поиска по русскому тексту в коде по умолчанию включена мультиязычная модель эмбеддингов (`paraphrase-multilingual-MiniLM-L12-v2`); при первом запуске она может скачаться автоматически. Настроить модель можно через переменную `RAG_EMBEDDING_MODEL` в `.env` (см. `backend/.env.example`).

### Бэкап и восстановление вселенной

- **Скачать архив вселенной:** `GET /api/universes/{id}/backup` — возвращает zip (БД + файлы + метаданные).
- **Восстановить из архива:** `POST /api/universes/restore` — тело запроса: файл `.zip` (form-data `file`). Создаётся новая вселенная с данными из архива. Опционально можно передать `title` для имени.
