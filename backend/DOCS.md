## Аппаратные требования (из старого документа)

### Оперативная память (RAM) и выбор моделей

У вас **32 ГБ оперативной памяти**. Это хороший объём для локального запуска LLM.

#### Локальный запуск (Ollama Local)
- **7B–8B** (`llama3.1:8b`, `qwen2.5:7b`) — быстро, оптимально
- **14B** (`qwen2.5:14b`) — баланс качества и скорости
- **32B** — могут работать медленно

#### Облачные модели (Ollama Proxy / API)
- Размер модели не ограничен локальной RAM
- Можно использовать `deepseek-v3:671b` и другие большие модели
- Контекст (`num_ctx`) ограничивается только лимитами провайдера

### Настройка контекста (`num_ctx`)
- Ollama по умолчанию: 2048 токенов
- ARCHON передаёт `num_ctx` (4096–8192+)
- Приложение автоматически управляет историей сообщений

## Работа с базой данных

### Автоматические миграции
Приложение автоматически проверяет и обновляет структуру БД при запуске через Alembic. Нет необходимости вручную выполнять SQL-запросы.

### Создание новой миграции
```bash
cd backend
alembic revision --autogenerate -m "description"
alembic upgrade head
```

## Префиксное кэширование (Prefix Caching)

Приложение оптимизировано для эффективной работы с Ollama:
1. **Stateless:** Вся история отправляется в каждом запросе
2. **Фиксированный порядок промпта:** Статичный мир → Персонаж → Динамические упоминания
3. **Prefix Caching:** Ollama кэширует неизменяемую часть, пересчитывается только динамический блок
4. **Экономия:** 30–50% времени генерации
