# MANIFEST — ArchonSystem

> **Этот файл — единственный источник истины.** Все агенты ОБЯЗАНЫ прочитать его первым перед любой работой.

---

## Цель

**ArchonSystem** — fullstack-приложение для worldbuilding'а, ориентированное на писателей-фантастов. Позволяет создавать и управлять вселенными, персонажами, локациями, главами, таймлайнами, фракциями, технологиями, артефактами, космосом. Глубокая интеграция с LLM для генерации контента, AI-чата с персонажами через RAG (векторный поиск ChromaDB), AI-критика и генерации изображений.

## Принципы

- **Постоянное самоулучшение** — система должна итерацию за итерацией становиться красивее, производительнее, современнее и элегантнее
- **Чистая архитектура** — слоистая: API → Services → Repositories → Models. Каждый слой знает только о нижнем
- **Изоляция данных** — отдельная SQLite база на каждую вселенную
- **LLM помогает, не царствует** — AI генерирует, но автор утверждает
- **Local-first** — всё работает локально (Ollama), облако опционально (OpenRouter, DeepSeek)
- **RAG с уровнями знаний** — персонажи знают только то, что им положено знать
- **Именование** — файлы агентов в kebab-case, без номеров-префиксов
- **Изоляция записи** — каждый агент пишет только в свои файлы
- **Тесты обязательны** — новый код = новые тесты (pytest / vitest)

- Агенты не удаляют MANIFEST.md и не подменяют цель
- Каждое действие документируется в отчёте
- Не делай предположений о структуре — сначала изучи
- Не ломай существующие тесты без обоснования
- Не плоди дубликаты — проверяй существующие файлы

## Рекомендуемые роли

Meta-agent создаёт роли на основе этой таблицы. Может объединять, разделять или добавлять новые.

| Роль | Функция |
|------|---------|
| **Backend-разработчик** | Развитие API, сервисов, моделей, миграций, RAG pipeline |
| **Frontend-разработчик** | Развитие React UI, новые страницы и компоненты, UX/дизайн |
| **AI-инженер** | Улучшение LLM-интеграции, промпты, контекст, генерация, AI-критик |
| **Тестировщик** | Покрытие тестами (pytest, vitest), E2E-тесты, регрессии |
| **Оптимизатор** | Производительность, рефакторинг, устранение мёртвого кода |
| **Документатор** | Актуализация docs/, inline-документация, CHANGELOG |
| **Контент-мейкер** | Генерация интересных текстов о развитии проекта для соцсетей |

## Правила владения файлами

| Файл / папка | Владелец (запись) | Остальные |
|---|---|---|
| `MANIFEST.md` | только автор | чтение |
| `agents/*.md` | meta-agent | чтение |
| `state/index.md` | meta-agent | чтение |
| `state/[имя].md` | только агент с этим именем | чтение |
| `backlog/index.md` | meta-agent | чтение |
| `backlog/[имя].md` | только агент с этим именем | чтение |
| `reports/summary.md` | meta-agent | чтение |
| `reports/[имя]/*.md` | только агент с этим именем | чтение |
| `backend/` | universal-agent (в роли, с тестами) | чтение |
| `frontend/` | universal-agent (в роли, с тестами) | чтение |
| `docs/` | universal-agent (в роли Документатор) | чтение |

**Правило:** если два агента могут работать одновременно — они **никогда** не пишут в один файл.
